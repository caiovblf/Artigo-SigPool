SigPool

Estudo prático sobre a fragilidade de assinaturas criptográficas no ecossistema de ativos digitais

O Problema: A Reutilização do Nonce no ECDSA

O Algoritmo de Assinatura Digital de Curva Elíptica (ECDSA), amplamente utilizado em redes de blockchain como o Bitcoin (sobre a curva secp256k1), possui uma dependência crítica: a geração de um valor aleatório e de uso único para cada assinatura, conhecido como nonce (k).

Se um mesmo valor de k for reutilizado para assinar duas mensagens ou transações diferentes utilizando a mesma chave privada, ocorre uma falha matemática grave. Um observador externo que tenha acesso a ambas as assinaturas válidas pode facilmente calcular o nonce e, consequentemente, extrair a chave privada correspondente de forma quase instantânea.

Relevância para o Ecossistema

No ecossistema de criptomoedas, a chave privada representa a propriedade soberana e irreversível dos fundos. Diferente de sistemas bancários tradicionais, não existe uma entidade central para estornar transações caso uma chave seja comprometida.

O grande perigo dessa vulnerabilidade reside no fato de que as transações continuam sendo matematicamente válidas. Elas entram na mempool, passam pelas verificações padrão dos nós e são incluídas em blocos sem levantar alertas imediatos nos validadores comuns. No entanto, a segurança do endereço foi completamente violada nos bastidores, permitindo que atacantes drenem todo o saldo restante.

A Solução Proposta: O Ambiente SigPool

Para mitigar o desconhecimento técnico e fornecer uma plataforma de testes segura, foi desenvolvido o SigPool: um laboratório didático local focado em simular e auditar essa falha clássica sem expor fundos reais à rede principal (mainnet).

O projeto simula um ambiente completo em miniatura:

  • Geração de Cenários Controlados: Permite injetar intencionalmente a falha de reutilização de nonce para fins de estudo (nonce-reuse), além de testar comportamentos normais (safe) e políticas de gastos como CPFP (Child Pays for Parent).
  • Auditor de Nonce Independente: Modulou-se um componente capaz de agrupar assinaturas públicas e flagrar instantaneamente a repetição do parâmetro r, recuperando a chave privada de teste de forma automatizada.
  • Isolamento de Camadas: Demonstra que a validação de formato na mempool e a descoberta de chaves comprometidas por auditoria são processos ortogonais, reforçando boas práticas de desenvolvimento de carteiras e infraestrutura de nós.

Como Executar a Demonstração

O laboratório funciona localmente de forma simples através de comandos de terminal:

# Clonar e entrar na pasta do projeto
cd project

# Instalar dependências necessárias
python -m pip install -r requirements.txt

# Executar o cenário de vulnerabilidade
python -m sigpool run --scenario nonce-reuse --seed 42
