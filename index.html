The Problem: ECDSA Nonce Reuse

The Elliptic Curve Digital Signature Algorithm (ECDSA), heavily utilized in blockchain networks (specifically over the secp256k1 curve), relies on a critical parameter: a cryptographically secure, single-use random value known as the nonce (k).

A catastrophic mathematical failure occurs if the exact same nonce k is used to sign two different transactions with the same private key. By isolating the signatures, an external auditor (or attacker) can calculate the nonce and subsequently recover the private key.

SigPool Architecture & Test Suite Insights

SigPool is not a wallet and does not interact with live networks. It is a controlled environment designed to demonstrate that network-level validation is orthogonal to cryptographic compromise. Our automated test suite guarantees the integrity of these simulations:

Mempool Simulator

  • Accepts mathematically valid signatures.
  • Rejects modified payloads (tampered test).
  • Rejects conflicting UTXO spends (double-spend test).

Block Assembler

  • Respects block weight limits (Max 1500w).
  • Prioritizes highest feerate.
  • Evaluates packages keeping parent-child dependencies (CPFP test).

Nonce Auditor

  • Independent background process.
  • Flags duplicate r values.
  • Recovers private keys and verifies them against victim public keys.

The Paradox of Validity

The most dangerous aspect of this vulnerability is demonstrated by the mempool output. The mempool accepted 2 transactions and the block assembler successfully packed them into a block weighing 1700 weight units with an average feerate of 0.88 sats/vB.

Because the signatures are mathematically perfect, traditional node consensus rules are fully satisfied. The network remains entirely unaware that the owner's private key has been exposed to the public through the auditor log, proving that strong consensus rules cannot protect against weak cryptographic implementations.
